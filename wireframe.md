# Wireframe UX — Landing page Restaurant 5 étoiles

Ce wireframe décrit la structure fonctionnelle (sans direction artistique) selon le parcours **Concierge Journey**.

## 1) Structure globale

```text
[HEADER STICKY]
Logo | Expériences | Menus | Le lieu | [CTA Réserver]

[HERO]
Promesse + phrase de valeur
[CTA primaire: Réserver] [CTA secondaire: Découvrir]

[SECTION: CHOIX D'EXPÉRIENCE]
[Carte Dîner Signature] [Carte Dégustation] [Carte Célébration] [Carte Privatisation]

[SECTION: PREUVES]
Distinctions | Avis premium | Garanties de service

[SECTION: MENUS & PHILOSOPHIE]
Colonne A: Vision culinaire
Colonne B: Infos pratiques/allergènes

[SECTION: LE LIEU]
Espaces | Horaires | Accès

[SECTION: RÉSERVATION GUIDÉE]
Étapes 1→5 + mini formulaire
Cas alternatif si complet: liste d'attente

[FOOTER]
Contact concierge | FAQ courte
```

## 2) Logique de flux UX

- Flux rapide: Hero → Réserver.
- Flux découverte: Hero → Expériences → Preuves → Réserver.
- Flux événement: Expériences/Privatisation → Réservation + note personnalisée.

## 3) Règles UX

1. Chaque section doit proposer une action claire.
2. Le CTA "Réserver" reste disponible en permanence.
3. Les informations critiques (horaires, allergies, accès) doivent être visibles sans recherche.
4. Les preuves de prestige apparaissent avant le formulaire pour renforcer la confiance.
